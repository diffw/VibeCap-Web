## E. Execution Strategy（强制）

1. 默认先跑受影响链路的 integration 测试（最小范围）
2. 若改动风险升高，再扩大到更大范围（跨模块、异步链路、核心能力）

`xcodebuild` 最小范围示例（`-only-testing`）：

```bash
xcodebuild \
  -project {PROJECT_NAME}.xcodeproj \
  -scheme {SCHEME_NAME} \
  -destination 'platform=macOS' \
  -only-testing:{TEST_TARGET}/SomeIntegrationTests \
  test
```

`xcodebuild` 全量 integration 示例：

```bash
xcodebuild \
  -project {PROJECT_NAME}.xcodeproj \
  -scheme {SCHEME_NAME} \
  -destination 'platform=macOS' \
  test
```

## F. Async/Timing Rules（强制）

- 优先使用 `async/await`
- 允许使用 `XCTestExpectation`
- 每个异步等待必须设置明确 `timeout`
- 禁止把固定 `sleep` 作为主同步机制

## G. Isolation & Cleanup（强制）

每个测试用例必须确保可重复执行与状态隔离，至少覆盖以下清理项（按实际触达启用）：

- `UserDefaults` 清理（测试 key 空间）
- 临时目录/临时文件清理
- `NotificationCenter` observer 清理
- 单例或全局状态复位（若测试触达）

## H. Real vs Mock Boundary（强制）

- 链路内模块：使用真实实现（真实交互）
- 外部依赖：允许 mock
  - 网络请求
  - 文件系统
  - 系统 API
  - 第三方服务
  - 系统时间
