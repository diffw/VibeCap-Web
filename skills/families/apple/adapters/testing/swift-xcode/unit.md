## Code Patterns

### Basic Test Structure
```swift
import XCTest
@testable import {PROJECT_NAME}

final class SomeModuleTests: XCTestCase {
    func test_functionName_givenCondition_expectedResult() {
        // Arrange
        let input = ...

        // Act
        let result = SomeModule.function(input)

        // Assert
        XCTAssertEqual(result, expectedValue)
    }
}
```

### Mock / Stub Patterns
- Extract protocols for services; inject test doubles. Place mocks in `{MOCKS_DIR}/`.
- For time-dependent logic, pass a `DateProvider` protocol and stub `now`.
- For purchase/entitlement logic, use mock service implementations to simulate pro/free states.

### Setup & Teardown
- Prefer `setUpWithError()` / `tearDownWithError()`; keep state isolated per test.
- Reset shared state (UserDefaults, singletons) in tearDown if any test mutates them.
