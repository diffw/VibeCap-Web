## Code Patterns

### Basic Test Structure
```swift
import XCTest

final class SomeFlowE2ETests: XCTestCase {
    private let app = XCUIApplication()

    override func setUpWithError() throws {
        continueAfterFailure = false
        app.launchArguments += ["-UITestMode", "1"]
        app.launch()
    }

    func test_journey_someFlow_happyPath() {
        // Step 1: trigger action
        app.buttons["action-button"].tap()

        // Step 2: verify result
        XCTAssertTrue(app.images["result-image"].waitForExistence(timeout: 5))

        // Step 3: complete flow
        app.buttons["confirm-button"].tap()
        XCTAssertTrue(app.staticTexts["success-message"].waitForExistence(timeout: 2))
    }
}
```

## Platform-Specific Considerations

- macOS E2E tests run on the host Mac. Ensure screen capture permissions are granted for CI runners.
- Cover critical flows: core feature flow, purchase flow, onboarding flow.
- For keyboard shortcut flows, use `app.typeKey("n", modifierFlags: .command)`.
- Avoid long sleeps; prefer `waitForExistence` with short timeouts.
- Menu bar interactions can be tested via `XCUIApplication.menuBarItems`.
