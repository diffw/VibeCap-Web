## Code Patterns

### Basic Test Structure
```swift
import XCTest

final class SomeFeatureUITests: XCTestCase {
    private let app = XCUIApplication()

    override func setUpWithError() throws {
        continueAfterFailure = false
        app.launch()
    }

    func testFeatureDisplaysCorrectly() {
        app.buttons["action-button"].tap()
        XCTAssertTrue(app.staticTexts["result-label"].waitForExistence(timeout: 2))
    }
}
```

### Mock / Stub Patterns
- Pass launch arguments for deterministic data: `app.launchArguments += ["-useMockData", "1"]`.
- Use `app.launchEnvironment` to fix locale for consistent assertions.

### Setup & Teardown
- Disable animations if flakiness appears: `app.launchArguments += ["-ApplePersistenceIgnoreState", "YES"]`.

## Platform-Specific Considerations

- macOS UI tests run on the host Mac (no simulator). Windows may need explicit activation: `app.activate()`.
- Add accessibility identifiers to SwiftUI views: `.accessibilityIdentifier("element-id")`.
- macOS supports hover, right-click, and keyboard navigation — test these where relevant.
- For menu bar app testing, use `XCUIApplication(bundleIdentifier:)` if needed.
- Keep flows short; prefer behavior assertions (visibility, text, toggles) over layout pixel checks.
